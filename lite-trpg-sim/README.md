# 轻量级跑团模拟器

一个本地运行的单机文字冒险 / 轻量 TRPG 模拟器。  
你可以把它理解成“在浏览器里游玩的单人跑团故事机”：

- 选择故事包
- 创建角色
- 阅读叙事
- 做出选择
- 进行掷骰检定
- 承受后果并走向不同结局

## 快速开始

### macOS 推荐方式

在项目根目录直接双击：

- `launch_game.command`

它会自动：

1. 启动本地后端
2. 启动前端页面
3. 在浏览器中打开游戏

### 命令行方式

如果你更习惯用终端，在项目根目录执行：

```bash
python3 launcher.py
```

然后在浏览器访问：

- `http://localhost:5173/index.html`

### 开始游玩的步骤

1. 选择故事包
2. 输入角色名
3. 选择职业
4. 点击“开始冒险”
5. 阅读剧情并选择行动
6. 根据检定、资源变化和事件结果继续推进

开始界面会显示当前故事包的简短说明、时长和风格标签。  
不同故事也可以把一些通用资源名换成更符合题材的叫法，例如 `摩拉`、`厄兆`、`侵蚀`。

## 现在能玩什么

当前内置 3 个故事包：

- `灰织边陲：腐月下的誓言`
  - 黑暗奇幻、边陲村庄、邪教、墓穴、腐化与生存压力
- `提瓦特：潮汐下的无名星灯`
  - 提瓦特冒险章节，偏调查、潜入、遗迹探索与角色登场
- `Demo: 灰烬哨站的封匣`
  - 一段简短的夜行任务，用来同时体验系统能力与项目的目标手感
  - 内含一个 Demo 专用的作弊职业，方便快速验收系统功能
  - 现在也明确展示 `V2.0` 的新增系统点，例如技能检定、限时状态、状态门槛动作与状态驱动难度变化
  - 开场包含一个可选的“技能试探”段，用来显式测试 `属性 + 技能` 的四种基础组合

每个故事包都支持：

- 创建角色
- 检定与分支推进
- 状态、物品、资源变化
- 失败、受伤、腐化或不同结局

## 一局游戏大概会经历什么

一次游玩通常会包含：

- 开场导入
- 职业选择
- 若干调查、探索或交涉场景
- 至少一个高风险事件或遭遇
- 若干检定与资源消耗
- 一个或多个可能结局

同一个问题通常不止一种解法。  
你可能靠战斗、潜行、调查、拖延、交涉或物品使用解决问题。

## 规则速览

### 角色

- 你会选择一个职业，并输入角色名开始冒险。
- 当前角色拥有这些基础属性：
  - `力量`
  - `敏捷`
  - `洞察`
  - `意志`
  - `交涉`
- 某些故事还会在属性之上提供一小组 `技能`，例如潜行、警觉、坚忍之类。
- 职业会影响：
  - 初始属性
  - 初始技能
  - 初始物品
  - 某些行动的优势方向

### 关键资源

角色当前会管理这些主要资源：

- `生命`
- `护盾`
- `腐化`
- `货币`

这些资源会影响你能否继续冒险、承受风险或走向某些结局。

### 检定

本作当前采用轻量 `d20` 检定：

- `d20 + 修正 >= 难度`

修正可能来自：

- 属性
- 技能
- 职业特性
- 物品
- 状态
- 场景条件

游戏界面会尽量告诉你：

- 掷骰结果
- 总修正
- 成功或失败
- 成功或失败的关键原因

### 状态、伤害与后果

- 你可能获得各种状态，例如受伤、惊惧、污染、祝福等。
- 部分状态会显示剩余回合数，说明它们会在若干轮后自然结束。
- 伤害不只是简单扣血：
  - 可能先被护盾吸收
  - 可能受到抗性或易伤影响
  - 某些情境还会附带额外后果
- 很多选择不会立刻结束故事，但会改变后续路线、资源或结局。

### 遭遇

有些场景不是单一步骤，而是“遭遇”：

- 可能是战斗
- 可能是潜入
- 可能是追逐
- 可能是交涉或拖延

遭遇通常会把风险、资源和选择压得更紧，让你在短时间内连续做判断。

## 存档与继续

- 支持 `5` 个手动存档槽位
- 支持自动存档
- 支持导入 / 导出存档
- 刷新页面后仍可继续最近进度
- 存档、读档、导入、导出的结果会直接显示在顶部槽位状态栏中
- 导入失败时会尽量指出是文件格式问题还是存档结构问题
- 行动执行失败时会在行动区下方直接显示错误信息，而不是打断式弹窗

如果你喜欢尝试不同路线，建议多用手动存档槽位分支保存。

## 常见问题

### 我需要一直开着终端吗？

需要。启动器运行时，本地前后端服务也在运行。关闭终端后，游戏页面会失去服务支持。

### 如果又看到“检测到错误启动方式”怎么办？

优先直接刷新当前标签页一次。  
如果仍然出现，请把警告卡片里显示的 `protocol` 和 `href` 一起反馈出来，这能帮助定位是浏览器误恢复了旧页面，还是当前标签确实被当成了 `file://` 页面。
当前启动器也已经尽量降低这个问题的概率：

- 前端静态资源使用禁缓存响应头
- 每次启动都会附带新的启动查询参数

这样可以减少 Safari 复用旧页面快照的概率。

### 我只能在浏览器里玩吗？

是。当前版本是“本地服务 + 浏览器界面”的形式。

### 这个项目以后能换故事背景吗？

可以。这正是它的设计目标之一。新的故事通常通过新增故事包接入，而不需要重写整个前端界面。
