# 工作原则

后续在本工作区内进行《斗罗大陆II绝世唐门》同人写作设定整理时，默认遵循以下原则，不再重复确认：

1. 这是“轻量防出戏设定库”，不是完整百科。
2. 主要依据工作区中的原文；使用外部资料时，必须再三确认不能和原文冲突，不脑补。
3. 只提取高频、关键、会影响同人是否出戏的设定。
4. 对不确定、不关键、太细碎的信息，宁可省略。
5. 少做解释，优先直接创建或更新文件。
6. 输出尽量短，适合快速检索，避免冗长大段落。
7. 所有目录和文件名必须是中文。
8. 不要为了“全面”而扫描和记录所有细节。
9. 优先服务后续同人写作，而不是资料整理本身。
10. 当《斗罗大陆II绝世唐门》与《斗罗大陆》存在冲突时，以《斗罗大陆II绝世唐门》为准。
11. 未经用户明确授权，所有操作不得超出当前工作区范围。
12. 原文与参考资料目录现为 `00原文材料+参考资料/`，后续默认从该目录读取资料。
13. 重要辅助资料源为本地缓存的百度百科《斗罗大陆II绝世唐门》词条 HTML：`00原文材料+参考资料/斗罗大陆II绝世唐门（唐家三少所著异世大陆类玄幻小说）_百度百科.html`。
14. 次要辅助资料源为本地缓存的百度百科《斗罗大陆》词条 HTML：`00原文材料+参考资料/斗罗大陆（唐家三少所著的长篇玄幻小说）_百度百科.html`。
15. 百度百科缓存文件仅作辅助校验与补充检索，不能覆盖工作区中的原文信息。
16. 若百度百科缓存文件与工作区原文存在冲突，仍以《斗罗大陆II绝世唐门》原文优先，其次才参考《斗罗大陆》原文。
17. 如果百度百科 HTML 缓存文件后续检索不便，可以先将其完整提取为保留原有信息与逻辑结构的 Markdown 文件，再基于该 Markdown 文件检索。
18. 执行 HTML 转 Markdown 时，应尽量完整保留信息，不随意删改，不打乱原有逻辑结构。
19. 当前已生成可检索的提取版 Markdown，后续优先检索提取版 Markdown，必要时再回看对应 HTML 原件。
20. 提取版 Markdown 仅为检索辅助层，不替代原始 HTML，也不高于工作区原文。
21. 对于长篇的非控制信息文本，默认应写入工作区内最合适的文档，而不是直接在聊天框大段输出。
22. 完成长篇文本写入后，只需在聊天中简要告知文件路径与用途，除非用户明确要求在聊天框全文展示。
