# Treasure Codex

这个子目录用于收录**具体宝物条目**。

上层文件负责说明宝物体系的规则、边界与分类；  
这里负责存放可直接用于设定与创作的具体宝物。

## 这个子目录的结构

本目录采用以下结构：

- 具体宝物条目直接放在本目录下，以 `.md` 文件存在
- 每件宝物如有概念图、示意图或其他辅助材料，统一放在 `assets/` 下对应的同名目录中

也就是说：

- 条目正文是主入口
- 条目资产是条目的辅助材料
- 外部文件引用宝物时，应优先引用宝物名称与条目本身

## 这个子目录收什么

这里收录的是已经可以被单独命名、单独描述、单独调用的宝物条目，包括但不限于：

- 具体器物型宝物
- 具体天然宝物
- 具体异宝
- 以及其中带有本命、战略等特殊属性的宝物条目

## 这个子目录不收什么

这里不收：

- 抽象规则
- 品级体系说明
- 契合、认主、温养的总纲性讨论
- 普通材料、普通丹药、普通耗材
- 尚未成型的零散灵感

这些内容应放在上层文件中，或待成熟后再进入条目库。

## 使用原则

- 上层文件讲规则，这里存条目
- 同一件宝物只保留一个主条目，避免重复收录
- 具体宝物条目应服从上层文件已经定下的边界与规则
- 条目应优先追求**可调用性**，而不是百科式穷尽
- 条目应优先写清**当前可调用状态**，历史变化只补关键节点

## 条目模板

具体条目写法与字段说明，见：

- `entry-template.md`

包括：

- 条目字段如何组织
- 同一文件中多个宝物条目如何排列
- 条目资产如何与正文对应使用

条目不要求写满，  
但必须项应尽量齐全。

## 命名规则

- 一件宝物只保留一个主名称
- 若有别称，可在正文中记录
- 条目文件名应尽量直接使用主名称
- 该宝物的资产目录名应与条目文件主名称保持一致
- 避免同物多名、同名多物或条目名与资产目录名不一致

## 阅读顺序

建议按以下顺序阅读：

1. 先看本 README
2. 再看 `entry-template.md`
3. 然后按宝物名称进入具体条目
4. 若该条目已有配套资产，再进入 `assets/` 下的对应目录查看

## 后续扩展

后续可根据创作需要继续扩展本目录。  
但无论如何扩展，都应优先保证：

- 条目入口清楚
- 资产归属明确
- 命名稳定一致
- 不重复收录
- 条目可直接用于小说写作
