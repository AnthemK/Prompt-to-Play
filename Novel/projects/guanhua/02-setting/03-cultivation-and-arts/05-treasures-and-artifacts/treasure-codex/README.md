# Treasure Codex

这个子目录用于收录**具体宝物条目**。

上层文件负责说明宝物体系的规则、边界与分类；  
这里负责存放可直接用于设定与创作的具体宝物。

## 这个子目录收什么

这里收录的是已经可以被单独命名、单独描述、单独调用的宝物条目，包括但不限于：

- 具体器物型宝物
- 具体天然宝物
- 具体异宝
- 以及其中带有本命、战略等特殊属性的宝物条目

## 这个子目录不收什么

这里不收：

- 抽象规则
- 品级体系说明
- 契合、认主、温养的总纲性讨论
- 普通材料、普通丹药、普通耗材
- 尚未成型的零散灵感

这些内容应放在上层文件中，或待成熟后再进入条目库。

## 使用原则

- 上层文件讲规则，这里存条目  
- 具体条目应服从上层文件已经定下的边界与规则  
- 条目归类时，应优先遵循上层文件已经定义的分类方式，而不要在条目库中另起一套体系  
- 当前阶段不预设固定分类树，后续可按创作需要逐步扩展  
- 同一件宝物只保留一个主条目，避免重复收录  
- 具体宝物条目应优先追求**可调用性**，而不是百科式穷尽  
- 条目应优先写清**当前可调用状态**，历史变化只补关键节点  

## 条目模板

具体条目写法与字段说明，见：

- `entry-template.md`

条目不要求写满，  
但必须项应尽量齐全。

## 层级说明

后续收录具体宝物时，应兼容两种常见情况：

### 1. 一个文件中收录多个宝物

这种情况下，应先有该文件的总标题，  
再在文件内部依次展开多个具体宝物条目。

若同一文件中收录多个宝物，建议在文件开头加入目录。  
若需要保证跳转稳定，优先使用**显式锚点**，而不要完全依赖自动标题锚点。

### 2. 一个文件只收录一件极重要宝物

这种情况下，该宝物本身可以直接作为该文件的核心对象展开，  
不必为了统一格式而强行降级。

无论采用哪一种写法，都应遵守两条原则：

- **同一文件内部的层级必须统一、清楚、稳定**
- **若同一文件中收录多个宝物，建议各条目尽量保持字段顺序一致**

## 命名规则

- 一件宝物只保留一个主名称  
- 若有别称，可在正文中记录  
- 条目命名尽量稳定，避免同物多名导致混乱  

## 后续扩展

后续可根据创作需要，在本目录下逐步建立更细的内部结构。  
但无论如何扩展，都应优先保证：

- 查找方便
- 命名统一
- 不重复收录
- 条目可直接用于小说写作
