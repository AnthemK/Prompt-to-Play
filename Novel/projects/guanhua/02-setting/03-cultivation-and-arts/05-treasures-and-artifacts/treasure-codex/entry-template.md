# Entry Template

本文件给出 `treasure-codex/` 中**具体宝物条目**的推荐模板。

它的作用是保证条目：

- 清楚
- 可调用
- 便于扩写
- 便于在同一文件中并列多个宝物
- 能兼容后续剧情推进中的状态与归属变化

它不是强制格式。  
不要求每条都写满，  
但**必须项应尽量齐全**，建议项尽量补足。

## 使用说明

后续写具体宝物时，可根据文件承载方式灵活调整层级：

- 若一个文件中收录多个宝物，则每个宝物条目使用统一层级展开  
- 若一个文件只写一件极重要宝物，则该宝物可直接作为文件核心展开  

无论采用哪一种写法，都应遵守一条原则：

> **同一文件内部的层级必须统一、清楚、稳定。**

此外，宝物条目不应被理解为完全静态的档案。  
若某件宝物会随着剧情推进而发生重要变化，则应优先保证：

- 当前情况清楚
- 关键变化可追踪
- 不因反复改写而失去条目稳定性

若同一文件中收录多个宝物，  
建议在文件开头加入目录。  
若需要保证跳转稳定，优先使用**显式锚点**。

## 字段分层

### 必须

- **名称**：这件宝物的主名称，以后统一引用以此为准。  
- **基本类型**：器物型宝物 / 天然宝物 / 异宝。  
- **核心作用**：用最短的话说明这件宝物最主要能做什么。  

### 建议

- **特殊属性**：本命宝物 / 战略重宝 / 无。  
- **大致品级**：按天地玄黄、上中下给出大致层次。  
- **适配对象**：更适合什么人、什么修途、什么体系或什么组织。  
- **限制与代价**：使用门槛、反噬、负担、环境要求、不宜轻启之处。  
- **来历或背景**：它从何而来，为什么重要。  
- **当前状态**：完整 / 残缺 / 封存 / 受损 / 未认主等。  
- **当前归属**：当前由谁持有、封存、掌控，或是否无主、失落。  

> 其中 **当前状态** 与 **当前归属** 强烈建议填写。  
> 对很多宝物而言，这两项会直接影响其现实意义。

### 可选

- **别称**：旧名、外号、俗称。  
- **外观特征**：便于增强辨识度与画面感。  
- **认主情况**：是否已认主，认主深浅如何。  
- **温养情况**：是否长期温养，是否已养出明显呼应或偏性。  
- **与组织或传承的关系**：尤其适合本命宝物与战略重宝。  
- **特殊使用条件**：例如依托某地脉、某阵法、某功法、某血脉。  
- **已知弱点 / 克制方式**：适合对抗性强的宝物。  
- **衍生影响**：例如是否影响心性、秩序、组织结构或外部争夺。  
- **关键变化记录**：用于简要记录归属、状态、认主、用途等方面的重要变化。  

> **关键变化记录** 只记录会显著影响宝物理解与调用的重要变化，  
> 不必事无巨细地写成剧情流水账。
