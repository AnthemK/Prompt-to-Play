# Treasure Codex Assets

这个子目录用于存放 `treasure-codex/` 中各个宝物条目的**配套资产**。

这里的资产是条目的辅助材料，  
不是条目本身，也不替代正文定义。

## 这个子目录的职责

本目录只负责两件事：

- 按宝物名称归档与该条目直接相关的辅助材料
- 为阅读者和协作者提供稳定、可追溯的资产入口

更具体地说：

- 宝物“是什么”，看上一级的条目 `.md`
- 宝物“有哪些配套材料”，看本目录下对应的同名子目录

## 组织方式

本目录采用以下结构：

- `assets/` 下直接按宝物主名称建立子目录
- 每个子目录只服务一个宝物条目

例如：

- 条目文件：`../中和道碑.md`
- 资产目录：`./中和道碑/`

应保持：

- 条目主名称一致
- 条目文件名与资产目录名一致
- 一件宝物只对应一个资产目录

## 这里可以放什么

本目录可以放与具体宝物条目直接相关的辅助材料，例如：

- 概念图
- 示意图
- 细节图
- 结构草图
- 生成记录
- 与资产直接相关的简短说明文件
- 其他确实服务于该宝物条目的辅助材料

## 这里不应放什么

本目录不应放：

- 宝物主定义正文
- 不对应具体宝物的通用图片
- 与多个条目混用、却没有明确归属的临时材料
- 与该宝物关系很弱的散乱实验文件

这些内容应分别回到：

- 上一级条目文件
- 更上层的设定正文
- 或其他更适合的实验目录

## 使用原则

- 条目正文始终是主定义文件
- 资产只做辅助理解，不替代文字设定
- 外部文件引用宝物时，应优先引用宝物名称与条目本身，而不是直接引用资产
- 若资产与正文发生冲突，以正文条目为准
- 若某项资产只是临时实验结果，不应直接视为正式设定

## 命名规则

- 资产目录名应与对应条目文件主名称一致
- 资产文件名应尽量直观，优先体现用途而不是随意编号
- 同一目录下若有多个版本，应在文件名中明确区分版本或用途

例如：

- `concept-main.png`
- `concept-alt-01.png`
- `detail-inscription.png`
- `notes.md`

## 阅读顺序

建议按以下顺序阅读：

1. 先读上一级 `treasure-codex/README.md`
2. 再读对应宝物条目
3. 最后进入本目录下该宝物的资产子目录查看辅助材料

## 后续扩展

后续若资产数量变多，仍应保持以下原则不变：

- 一物一目录
- 正文优先
- 归属明确
- 命名稳定

若未来某件宝物的资产复杂到需要单独说明，可在该宝物资产目录内补一个局部 `README.md`，  
但不应反向替代上一级条目正文。
