# 01 Order Framework

这个目录处理 `05-factions-and-polities/` 之下的**总图层**。

这里不写具体势力档案，而写它们背后的共性框架。  
它主要回答：

- 这个世界里有哪些主要秩序类型
- 不同类型的势力通常如何成立、维持与扩张
- 它们通常能管什么、依赖什么、如何互动、为何被承认

换句话说：

- 这里写“共性”
- 不写“个案”

## 本目录负责什么

本目录主要用于：

- 为各类具体势力提供统一分类框架
- 说明势力之间常见的权力边界、资源关系与互动方式
- 说明地位、名分与承认如何在本世界中成立

如果要新建一个势力档案，通常应先阅读本目录。

## 本目录不负责什么

以下内容不在本目录展开：

- 某个具体宗门、世家、王朝、学宫或其他势力的详细设定  
  - 请去对应子目录
- 世界底层规则与总约束  
  - 见 `01-constitution/`
- 天地、道则、灵机、界域、飞升、天劫等本体问题  
  - 见 `02-cosmos-and-principles/`
- 修炼体系、功法、法术、法宝等修行问题  
  - 见 `03-cultivation-and-arts/`
- 地理格局、地图、历史时间线与时代变化  
  - 见 `04-world-layout-and-history/`

## 文件索引

- `order-types.md`
  - 说明本世界主要秩序载体的类型划分

- `authority-and-jurisdiction.md`
  - 说明不同类型势力通常能管什么、管到哪

- `dependency-and-resource-relations.md`
  - 说明不同类型势力通常依赖什么资源与关系网络

- `conflict-alliance-and-intervention.md`
  - 说明不同类型势力通常如何冲突、结盟与彼此介入

- `status-legitimacy-and-recognition.md`
  - 说明不同类型势力为何被承认、视为正统、附庸或僭越

## 建议阅读顺序

首次进入本目录，建议按以下顺序阅读：

1. 本 README
2. `order-types.md`
3. 其余四个文件按任务需要阅读
4. 再进入对应类型的具体势力目录

## 最后说明

这一层的作用，是为后续所有具体势力档案提供一个统一前提：

- 先理解框架
- 再建立个案

具体势力的辖地、结构、人物、资源与剧情功能，不在这里展开。
