# 01 Foundation

这个目录处理《观化》的立项问题。

先回答：

- 这本书卖什么
- 读者为什么会追下去
- 主题问题是什么
- 哪些研究是必要的

在这一步，不进入详细世界百科，不进入章节草稿。

这一层负责的是小说本体问题，而不是协作治理问题。

换句话说：

- 这里回答《观化》是什么
- 不回答 cache、命名、目录治理和任务执行规则

## 这个目录解决什么问题

如果你是新协作者，这一层主要回答：

- 《观化》到底想写什么
- 读者会因为什么继续读下去
- 主题和思想重心是什么
- 哪些研究方向有用，哪些暂时不用展开

## 文件索引

- `project-brief.md`
  - 项目总说明，回答“这本书是什么”
- `genre-promise.md`
  - 类型承诺，回答读者为什么会追下去
- `thematic-core.md`
  - 主题问题与思想重心
- `novel-style-principles.md`
  - 小说本体的文风原则与思想取向
## 建议阅读顺序

1. `project-brief.md`
2. `genre-promise.md`
3. `thematic-core.md`
4. `novel-style-principles.md`

## 给协作者的建议

如果你只需要快速上手：

- 先读本目录 README
- 再优先读 `genre-promise.md` 和 `thematic-core.md`

如果你正在做文风、主题、思想性的实验性写作，这一层必须先过一遍。

如果你要看外部参考与研究笔记，请去：

- `00-reference/README.md`

如果你要看协作规则、命名规则或写作执行规则，请回到：

- `docs/DOC_GOVERNANCE.md`
- `docs/NAMING_RULES.md`
- `docs/WRITING_EXECUTION_RULES.md`
