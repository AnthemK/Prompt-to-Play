# 01 Foundations

这个目录处理本文的本体立项问题。

这一层不写详细世界设定，不写人物小传，也不写章节推进。

它只回答四件事：

- 本文到底是什么样的小说
- 本文向读者承诺什么体验
- 本文真正想讨论什么问题
- 本文的文风、思想底色与审美边界是什么

## 这个目录解决什么问题

如果你是新协作者，这一层主要回答：

- 本文到底想写什么
- 为什么它适合写成长篇
- 读者会因为什么继续读下去
- 主题、思想与叙述重心是什么
- 文风上应靠近什么，避免什么

## 文件索引

- `project-brief.md`
  - 项目总说明。先看这里，建立对本文题材定位、核心问题与长篇驱动力的整体认识。
- `genre-promise.md`
  - 类型承诺。回答本文向读者承诺了哪些阅读体验、结构节奏与写作禁忌。
- `thematic-core.md`
  - 主题核心。回答本文真正要逼问的问题，以及生存、认知、秩序与政治的主题张力。
- `novel-style-principles.md`
  - 文风与思想原则。回答本文的语言气质、思想底色、命名要求与叙述立场。

## 建议阅读顺序

1. `project-brief.md`
2. `genre-promise.md`
3. `thematic-core.md`
4. `novel-style-principles.md`

## 给协作者的建议

如果你只需要快速上手：

- 先读本目录 README
- 再读 `project-brief.md` 与 `genre-promise.md`
- 最后用 `thematic-core.md` 和 `novel-style-principles.md` 校准主题与文风

如果你正在做世界设定：

- 先确认 `project-brief.md` 与 `thematic-core.md`
- 再进入 `02-setting/README.md`

如果你正在做人物、情节或正文：

- 先确认 `genre-promise.md` 与 `novel-style-principles.md`
- 再进入对应下层目录

这一层只提供“上位约束”，不提供世界百科、人物履历或剧情细纲。

如果你要看外部参考与研究笔记，请去：

- `00-reference/README.md`

如果你要看协作规则与执行规则，请去：

- `docs/DOC_GOVERNANCE.md`
- `docs/NAMING_RULES.md`
- `docs/WRITING_EXECUTION_RULES.md`
